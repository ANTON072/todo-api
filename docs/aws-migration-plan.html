リスク・注意点
⚠️ コネクション上限
Lambda高同時実行時にPostgreSQLの接続数が枯渇する可能性。低トラフィック時は問題なし。必要になれば RDS Proxy を追加。
⚠️ コールドスタート
VPC内Lambdaは2020年以降1秒未満。Aurora Serverless v2の最小0.5 ACUに設定すれば完全ゼロスケールを回避できる。
⚠️ 2038年問題
PostgreSQLのintegerは最大約21億 ≒ 2038年1月まで。長期運用するなら dueDate を bigint に変更することを検討。
