AWS に作られるリソース（6個）

🌐
VPC
仮想プライベートネットワーク
  • Lambda と RDS を同じネットワークに入れる「箱」
  • パブリックサブネット × 2（AZ を分けて冗長化）
  • NAT ゲートウェイなし → コスト削減
🛡️
セキュリティグループ
ファイアウォール設定
  • RDS は Lambda からの 5432 番ポートのみ許可
  • それ以外（インターネットなど）は全部ブロック
  • Lambda 自体はアウトバウンド全許可（外部 API を呼べる）
🐘
RDS PostgreSQL
db.t3.micro / PostgreSQL 16
  • データを永続保存するデータベース
  • db.t3.micro = 最小サイズ（コスト最小化）
  • ストレージ暗号化あり、パブリックアクセスなし
  • 接続パスワードは CDK が自動生成→ Secrets Manager に保存
λ
Lambda
Node.js 22 / 512MB / 30秒
  • Hono アプリを動かすサーバー（サーバーレス）
  • リクエストが来たときだけ起動 → 待機コストゼロ
  • 512MB・30 秒タイムアウト（better-auth の処理に余裕を持たせる）
🚪
API Gateway HTTP API
エンドポイント公開
  • インターネットからリクエストを受けて Lambda に渡す「入口」
  • HTTPS URL が自動で払い出される
  • CORS 設定あり（全オリジン許可）
🔐
Secrets Manager
シークレット × 2
  • DB パスワードをコードに書かずに管理する仕組み
  • Lambda 起動時に自動で取得して使う
  • 詳細は下の「シークレットの中身」参照

Secrets Manager に保存するシークレット

① DB シークレット（CDK が自動生成）

RDS インスタンスのパスワードを CDK が自動で生成・保存する。手動作業不要。

usernametodoapi（固定）
password自動生成されたランダム文字列
hostRDS のエンドポイント URL
port5432
dbnametodoapi

② アプリシークレット（デプロイ後に手動設定が必要）

CDK がシークレットの「箱」だけ作る。値はデプロイ後にAWSコンソールで設定する。

BETTER_AUTH_SECRETCDK が自動生成（変更不要）
RESEND_API_KEY⚠️ "REPLACE_ME" → Resend ダッシュボードの値に変更する
ALLOWED_ORIGINS⚠️ 空文字 → 許可するフロントエンドの URL を設定する

Lambda が起動したときの流れ（src/lambda.ts の改修内容）

コールドスタート
Lambda コンテナが初めて起動する
シークレット取得
Secrets Manager から DB パスワードとアプリシークレットを取得
2つのシークレットを並列で取得してレイテンシを最小化
URL 組み立て
DATABASE_URL を組み立てる
例: postgresql://todoapi:password@xxxx.rds.amazonaws.com:5432/todoapi?sslmode=require
リクエスト処理
以降のリクエストはキャッシュした値を使い回す（再取得しない）
Lambda コンテナが生きている間はコールドスタートのコストが発生しない

Phase 6 で追加したコマンド

# Lambda 用バンドルを作る（cdk deploy の前に必ず実行） pnpm build:lambda # CDK コマンドのショートカット（infra/ に cd しなくてよい） pnpm cdk synth # CloudFormation テンプレートを確認（AWSには何も作らない） pnpm cdk diff --profile ougi-aws # 現在のAWSとの差分を確認 pnpm cdk deploy --profile ougi-aws # 実際にAWSにデプロイ（Phase 7）
⚠️ Phase 6 はあくまで「設計図を書いた」段階。 AWSに実際にリソースが作られるのは Phase 7 の cdk deploy を実行したとき。
